1工作原理：GitHub 到双平台的自动发布机制

网站使用纯静态生成方案，无需服务器，整个发布链路如下：

各环节详解

① 本地写作：复制项目根目录的 article-template.md 模板到 content/articles/ 目录，重命名后填写文章内容。

② Git 推送：在终端执行 git push，将修改上传到 GitHub 远程仓库。推送一完成，GitHub 立即向 Netlify 和 Cloudflare Pages 发送 Webhook 通知。

③ 自动构建：两个平台分别拉取最新代码，在各自服务器上执行 npm run build。构建脚本 build.js 会：

  • 读取所有 .md 文章文件，提取 frontmatter 信息
  • 将 Markdown 正文转换为 HTML 页面
  • 生成首页文章列表（index.html）和各文章独立页面
  • 生成 RSS 订阅文件（feed.xml）

④ 全球上线：构建产物部署到各平台的 CDN 节点，通常 1–3 分钟内全球访客即可看到最新内容。两个平台完全独立构建，互不影响。

💡

一次 git push 自动触发两个平台同步更新，无需分别操作。

3发布新文章：操作步骤

1
复制文章模板并重命名

项目根目录中有现成的模板文件，直接复制到文章目录并重命名：

模板文件位置
D:\AIWorkplace\claude\personal-website\article-template.md
复制到文章目录，按以下格式重命名
D:\AIWorkplace\claude\personal-website\content\articles\YYYY-MM-DD-英文简称.md

命名示例：2025-08-01-patent-license-analysis.md

2
填写文章头信息（Frontmatter）

模板顶部已预置 frontmatter 结构，修改各字段内容即可：

模板预置内容
--- title: 文章标题 date: 2025-01-01 category: case summary: 在此填写首页卡片摘要，建议 50–120 字，概括文章核心观点。 link: "#" type: 案例评析 ---

各字段含义见第 4 节说明。

3
撰写文章正文

在 frontmatter 后面写正文内容，使用 Markdown 语法。用 ## 写一级章节标题，### 写二级标题，**文字** 加粗。详见第 6 节。

4
本地构建（可选，用于预览）

打开终端，进入项目目录，执行构建命令预览效果：

cd D:\AIWorkplace\claude\personal-website npm run build

构建成功会提示"✓ 构建完成：XX 篇文章"。用浏览器打开 index.html 即可本地预览。

5
提交并推送到 GitHub

执行以下命令，文章将在 1–3 分钟内自动上线两个平台：

git add content/articles/你的文件名.md git commit -m "添加新文章：文章标题" git push origin main
💡

推送后无需任何额外操作，两个平台会自动完成构建和发布。

6
验证上线

等待约 2–3 分钟后，打开 https://personal-website-6dn.pages.dev 确认新文章已出现在首页列表中，点击进入文章页面验证内容正确。

4Frontmatter 字段说明

字段 必填 说明 示例
title ✓ 文章标题，显示在卡片和文章页标题栏 专利权属争议期间放弃专利的责任
date ✓ 发布日期，格式 YYYY-MM-DD。首页文章列表按此日期倒序排列（最新在前） 2025-08-01
category ✓ 文章分类，决定标签颜色和筛选按钮归属。可选值见第 5 节 case
summary ✓ 首页卡片上显示的摘要文字，建议 50–120 字，清晰概括核心观点 以三个合同纠纷案例为切入点……
link ✓ 外部链接。如有原文 URL（如微信公众号）填入；无则填 # # 或 https://mp.weixin.qq.com/…
type ✓ 文章类型标注，显示在卡片右下角。自由填写 案例评析 / 工作论文 / 发表论文 / 读书笔记
⚠

frontmatter 必须以 --- 开头和结尾，字段名与值之间用英文冒号加空格 : 分隔。格式错误会导致文章无法正确解析。

6正文写作说明

标题层级

## 一、第一节标题 ← 一级章节（生成目录锚点） ### 1、子节标题 ← 二级小节 #### 更细的层级 ← 三级（少用）

常用格式

**加粗文字** ← 重要概念或结论 *斜体文字* ← 引用或强调 - 无序列表项一 - 无序列表项二 1. 有序列表项一 2. 有序列表项二 > 引用块：适合引用法条原文或判决原文 `行内代码` ← 法条编号、案件编号等

脚注

正文中引用处[^1]，另在结尾添加： [^1]: 脚注内容，如：《民法典》第五百六十二条。

嵌入图片

将图片文件放入项目根目录的 images/ 文件夹，然后在正文中用以下语法引用：

![图片说明文字](../images/文件名.png)

图片会自动渲染为带说明文字的居中图（<figure> 结构）。发布时需将图片文件一并 git add 提交。

git add images/文件名.png content/articles/文件名.md

插入表格

正文中可直接写 HTML 表格，构建系统会原样保留并应用预置样式：

<table class="data-table"> <caption>表格标题</caption> <thead> <tr><th>列一</th><th>列二</th><th>列三</th></tr> </thead> <tbody> <tr><td>数据</td><td>数据</td><td>数据</td></tr> </tbody> </table>
💡

HTML 块（表格、SVG、div 等）与前后段落之间须各空一行，否则会被误判为普通段落文字。

注意事项

  • 段落之间需要空一行，否则会被识别为同一段。
  • 中文标点（，。""）可直接使用，无需转义。
  • 法条引用建议用**《民法典》第X条**加粗，提升可读性。
  • 如文章引自微信公众号，可在结尾加"附：原文链接 URL"。

7常用命令速查

进入项目目录

cd D:\AIWorkplace\claude\personal-website

本地构建预览

执行构建（生成 HTML 文件）
npm run build

发布到 GitHub（触发两平台自动部署）

发布单篇新文章
git add content/articles/文件名.md git commit -m "添加新文章：文章标题" git push origin main
发布多个改动（如修改了 CSS、模板或多篇文章）
git add -A git status ← 确认暂存的文件清单 git commit -m "更新说明" git push origin main

修改已有文章

← 直接编辑对应的 .md 文件，然后： git add content/articles/对应文件名.md git commit -m "修改：文章标题（说明修改内容）" git push origin main

查看当前状态

git status ← 查看有哪些文件被修改或新增 git log --oneline -5 ← 查看最近 5 次提交记录
